Denby 리뉴얼 사이트

site 기획서(PDF)

- 빌레로이앤보흐 사이트 벤치마킹, 1인 프로젝트

- 버전 : PC(1400px), Tablet(1024px), Mobile(360px)

- 기간 : 24.03.29 ~ 24.04.24

- 리뉴얼된 웹사이트는 GSAP와 LENIS 라이브러리를 활용하여 부드러운 스크롤 효과를 적용함으로써, 시각적 효과를 더해 고급 브랜드 이미지를 강조했습니다.

기술 소개

이미지 위로 마우스를 올리면 영상이 재생됩니다.

- 메뉴 구현

: 메뉴 영역에 마우스를 올리면 서브 메뉴가 펼쳐지는 jQuery를 작성하였습니다. 헤더 및 하위 메뉴 영역에 마우스가 있는 동안 서브 메뉴가 사라지지 않도록 구현했습니다.

- parallex 효과 + 스크롤 시 배경 고정

: 메인 배너에 Parallex 슬라이드 효과와 GSAP의 ScrollTrigger를 사용한 배경 고정 효과를 적용하여, 웹 페이지를 더 생동감 있고 입체적으로 표현함으로써 사용자 경험을 향상시켰습니다.

- sticky 고정 효과

: 섹션의 상단에 GSAP의 ScrollTrigger sticky 효과를 사용하여, 오른쪽 요소들이 순서대로 올라오며 돋보이도록 적용했습니다.

- 스크롤 opacity 효과

: GSAP의 ScrollTrigger를 이용해 특정 영역에 진입하면 섹션의 불투명도와 배경 크기를 조절하여 주목도를 높였습니다.

코딩 도전기

: 오프라인에서 자주 접하던 브랜드의 웹사이트에 들어가 보니, 브랜드 이미지와 웹사이트의 디자인이 서로 어울리지 않는다는 느낌을 받았습니다. 다른 브랜드의 사이트를 비교하고, 부족한 부분을 분석해 보았습니다. 그 중에서도 빌레로이앤보흐라는 브랜드의 웹사이트는 고급스러운 이미지를 잘 전달하고 있었고, 판매 중인 제품들이 돋보이게 디자인되어 있었습니다. 그래서 그 사이트를 벤치마킹하기로 결정했습니다.

여러 라이브러리를 학습 중인 저로서, 해당 사이트에서 사용된 lenis와 gsap 라이브러리를 처음 접했을 때 다루는 데 어려움을 겪었습니다. 제이쿼리를 활용하여 비슷한 효과를 구현해 보았지만, 원하는 고급스러운 느낌을 표현하는 데에는 부족함이 있었습니다. 그래도 처음이라면 어쩔 수 없는 일이었고, gsap와 lenis 라이브러리에 대해 자료를 찾아가며 다양한 시도를 해보았습니다.

그렇게 해서 '스크롤 트리거'라는 것을 배우고 적용함으로써, 목표했던 고급스러운 웹사이트의 느낌을 살려내고 사용자 경험을 향상시킬 수 있었습니다. 이러한 노력 끝에 성공적으로 프로젝트를 마무리할 수 있었습니다.

Touslesjours
리뉴얼 사이트

site 기획서(PDF)

- 풀무원 사이트 벤치마킹, 1인 프로젝트

- 버전 : PC(1400px), Tablet(1024px), Mobile(360px)

- 기간 : 24.02.29 ~ 24.03.28

- 뚜레쥬르 브랜드의 '갓 구운 신선한 빵을 판매하는 베이커리' 이미지를 강조하기 위해 메인 배너에 동영상을 삽입하고, 신선한 빵을 준비하는 하루를 한 섹션으로 표현하였습니다.

기술 소개

이미지 위로 마우스를 올리면 영상이 재생됩니다.

- 메인 배너 동영상 배치

: 웹사이트 접속 시 제일 먼저 보여질 메인 배너에는 뚜레쥬르의 신선한 빵내음을 연상시키기 위해 빵을 굽는 동영상을 배치하였습니다.

- 각 섹션을 연결하는 앵커

: a태그의 링크와 data를 각 섹션의 name과 동일하게 설정하여, 메인 배너 버튼을 클릭하면 해당 섹션의 위치로 스크롤되도록 jQuery로 구현했습니다.

- 모바일 메뉴 구현

: 태블릿과 모바일 메뉴는 아코디언 메뉴로 구현되어 있으며, 다른 메뉴를 클릭하면 이전에 활성화된 메뉴는 자동으로 비활성화되도록 jQuery 코드를 작성했습니다.

- footer 계열사 사이트

: 푸터 영역의 계열사는 접혀있다가 클릭되면 펼쳐지도록 jQuery 코드를 작성했고, 메뉴 영역에서 마우스가 벗어나면 자동으로 접히도록 슬라이드 기능을 구현했습니다.

코딩 도전기

: 벤치마킹을 통해 뚜레쥬르의 사이트를 리뉴얼하는 것은 처음이었기 때문에 많이 막막했습니다. 뚜레쥬르 사이트를 분석한 결과, 마케팅을 적극적으로 하지 않고 있다보니 아이콘 및 이미지가 부족하여 벤치마킹 할 사이트와 유사하게 작업하기 어려움을 느꼈습니다. 영상과 아이콘이 부족했지만 무료 이미지 및 동영상, 아이콘을 제공하는 사이트에서적절한 자료를 찾아 적용했습니다.

특히, 2번째 섹션의 추천 제품을 코딩하는 과정에서는 피그마로 구상한 배치가 적용되지 않아 어려움을 겪었습니다. 다양한 방법을 시도해봤지만 pc 버전에서 적용될 fullpage 효과로 여백이 부족했습니다. 브랜드 가치를 높이고 추천 제품을 부각시키는 것이 목표였지만, 적절한 벤치마킹 사이트를 찾지 못해 아쉬웠습니다. 따라서, 일반 슬라이드 3개로 배치하는 대체 방법을 사용했습니다. 앞으로 도전할 리뉴얼 사이트에서는 적절한 벤치마킹 사이트를 찾거나 해당 영역의 특징을 잘 나타내는 효과를 찾아볼 것 입니다.

이외에도 fullpage 적용과 a태그에 섹션 상단으로 스크롤링하는 링크를 구현하는 과정에서도 어려움을 겪었습니다. 그러나 벤치마킹 사이트의 코드를 그대로 따라가는 것이 아니라 나만의 코드를 작성하며 실력이 향상되었다는 것을 느꼈습니다.

Starbucks 리뉴얼 사이트

site

- Starbucks 클론코딩

- 버전 : PC(1440px), Tablet(1024px), Mobile(360px)

- 기간 : 24.01.12 ~ 24.02.02

- 스타벅스 웹사이트의 TweenMax 라이브러리 애니메이션 효과를 클래스에 transition 속성을 이용하여 적용했습니다.

기술 소개

이미지 위로 마우스를 올리면 영상이 재생됩니다.

- 돋보기 아이콘 클릭 시 input 창 활성화

: 제이쿼리를 이용하여 header 상단의 돋보기 버튼을 클릭하면 검색어 입력란의 너비가 넓어지도록 조정했습니다.

- 스크롤에 따른 애니메이션 효과

: 제이쿼리를 이용하여 스크롤 위치를 감지합니다. 해당 영역의 애니메이션 될 요소에 'active', 'delay' 클래스를 추가함으로써 동작하는 애니메이션 효과를 구현해 보았습니다.

- 아코디언 메뉴와 버튼 회전 효과 구현

: 모바일 메뉴는 제이쿼리로 구현한 아코디언 메뉴이며, 메뉴가 펼쳐질 때 닫기 버튼이 회전하며 나타납니다.

- 수상내역 슬라이드 멈춤/재생 기능 구현

: 수상내역을 스와이퍼의 슬라이드로 삽입하여 자동 재생되도록 하고, 제이쿼리를 이용하여 멈춤/재생이 가능한 버튼을 제공했습니다.

코딩 도전기

: 스타벅스 페이지에서는 gsap 라이브러리 효과로 애니메이션을 구현하였지만 클론코딩 페이지에서는 active 클래스를 추가하여 애니메이션을 적용해 보았습니다.

클론 코딩을 진행하면서 웹페이지 구조를 작성하는 과정에서 해상도에 따라 적용될 화면 구성을 고려해야 하는 어려움이 있었습니다. 이러한 작업을 보다 쉽게 할 수 있도록 피그마를 통해 작성한 와이어프레임과 프로토타입이 HTML 구조 작성 시 많은 도움이 되었습니다. 또한, 공통적으로 사용되는 글꼴, 글꼴 크기, 여백, 위치 등을 파악하는 시간을 단축시켜 주어 클래스 사용, display, position의 이해도가 향상되었습니다.

또한, 제이쿼리 문법에 익숙하지 않아 애니메이션을 적용할 때 코드 작성이 어려웠습니다. JavaScript 문법이 설명된 책을 참고하고 적절한 이벤트를 찾아보고, 여러 웹사이트를 분석하여 효과와 사용된 코드를 이해하였습니다. 단계별로 작동할 단계를 기술하고 실행 및 수정하는 과정을 통해 코드를 완성했습니다.

클론 코딩을 하면서 제이쿼리 문법에 익숙해지는 과정에서 어려움을 겪었지만, 코드 분석을 통해 단계적으로 성공을 이끌어냈습니다.

영화 예매 사이트

site 기획서(PDF)

- 영화 예매 사이트, 1인 프로젝트

- 버전 : PC(1440px)

- 기간 : 24.04.22 ~ 24.05.28

- API를 사용하여 영화 정보를 지속적으로 업데이트 되도록 구축하였습니다. 최신 영화에 대한 정보를 제공하는 서비스로, 스프링 시큐리티를 사용해 로그인 인증 및 인가 기능을 구현하였습니다. 로그인된 사용자는 영화, 날짜, 좌석을 선택하여 티켓을 예매합니다.

기술 소개

- 로그인 화면

: 스프링 시큐리티를 이용하여 로그인 인증 및 인가 기능 구현하였습니다. 로그인 후 영화 예매 서비스를 제공합니다.

- 일정 및 좌석 선택 화면

: 타임리프와 자바스크립트를 사용해 선택한 영화 속성을 hidden 필드로 전달받습니다. jQuery의 datePicker를 이용해 달력을 삽입하고, RequestParam을 통해 날짜와 좌석 정보를 다음 페이지로 전달합니다.

- 예매 내역 확인 및 취소 기능

: Principal을 통해 로그인한 사용자의 예매 정보를 조회하여 화면에 출력합니다. 예매 취소 버튼을 누르면 PathVariable로 예매 번호를 전달받아 Service를 호출하여 티켓을 취소할 수 있는 기능을 제공합니다.

- 빠른 예매 기능 제공

: 사이트 우측 상단의 '빠른 예매'를 선택하면 Repository를 사용해 BoxOffice 전체 목록을 조회합니다. model을 통해 영화 목록 데이터를 전달하며, Top10 영화 목록이 왼쪽에 나열되고, 사용자는 달력과 좌석을 선택할 수 있습니다.

코딩 도전기

: 자바 언어를 공부하면서 어떻게 활용될지에 대한 궁금증이 많았었는데, 이번 프로젝트를 통해 자바의 원리를 깊이 이해할 수 있는 계기가 되었습니다.

API를 연동하여 속성을 끌어다 쓰는 과정을 통해서 웹 애플리케이션의 동작 원리도 익힐 수 있었습니다. 특히 웹페이지와 도메인을 연결해줄 컨트롤러, 타임리프로 HTML 을 구성하는 과정에서 문법 오류, 정보 전달 누락 등의 오류로 인해 많은 어려움을 겪었습니다.

풀기 어려울 것으로 예상했던 에러도 다양한 관점에서 바라보고 도전해보니 결국 문제를 해결할 수 있다는 것을 깨달았습니다.이러한 과정을 통해 개발자 도구를 더 친숙하게 사용할 수 있게 되었고, 에러에 대한 두려움을 극복할 수 있게 되었습니다.

Denby 쇼핑몰 기능 구현

준비중

- 온라인 쇼핑몰 구현, 1인 프로젝트

- 버전 : PC(1440px)

- 기간 : 24.05.07 ~

- 스프링 시큐리티를 사용하여 로그인 인증 및 인가를 구현하였습니다. 가입된 사용자에게 상품 주문, 주문정보 확인 및 취소 등의 기능을 제공합니다. 장바구니, 찜, 리뷰 기능을 구현 중이며, 관리자 등급 부여를 통해 화면 접근 제한 기능도 추가 적용할 예정입니다.

기술 소개

- 회원가입

: 약관동의 및 정보입력이 이루어집니다. 자바스크립트를 사용하여 약관에 동의하지 않거나 필수 정보를 입력하지 않으면 가입할 수 없도록 설정했습니다. Valid와 BindingResult를 통해 에러가 발생하면 메세지를 전달합니다.

- 마이페이지

: 로그인된 사용자의 정보를 포함하고 있습니다. 사용자 정보 수정이 가능하며, userForm을 통해서 유효성 검사를 진행합니다. 아이콘을 누르면 사용자 구매내역, 장바구니 상품 목록, 위시리스트, 작성한 리뷰로 이동하는 링크가 연결되어 있습니다.

- 상품 상세페이지

: [Add To Cart] 버튼과 [Buy] 버튼에는 hidden 속성으로 값 저장하여 데이터를 넘겨주는 자바스크립트를 구현하였습니다. 장바구니에는 타임리프의 th: 속성을 이용하여 Basket 테이블에 정보 저장, 주문은 RedirectAttributes를 통해 주문 정보 입력 화면에 정보를 전달합니다.

- 주문 정보 입력 화면

: ModelAttribute를 이용하여 매개변수를 받습니다. 상품정보, 수량, 사용자의 잔여 포인트를 화면에 표시합니다. 결제 금액이 10만원 이상 무료배송 적용, 보유 포인트를 사용하여 최종 결제금액에서 포인트 차감이 이뤄지는 자바스크립트가 적용되어 있습니다.

코딩 도전기

: 백엔드 프로젝트를 진행하면서 원인을 쉽게 알 수 없는 에러에 부딪혀 많은 어려움을 겪었습니다. 자바스크립트 함수 작성이나 타임리프 문법 오류처럼 명확한 원인이 보이는 경우도 있었지만, 그렇지 않은 경우에는 모든 코드를 일일이 점검해야 했습니다. 혼자서 프로젝트를 진행하다 보니 지치기도 하고, 과연 내가 이 문제를 해결할 수 있을지 의구심이 들기도 했습니다.

특히 프로젝트 초기에는 일주일 동안 에러를 수정하지 못한 적도 있었습니다. 선생님께 도움을 청했지만 해결하지 못했고, 정보처리 산업기사 공부할 때 조언을 주신 지인분께 도움을 요청해 문제를 해결했습니다. 원인이 복합적인 에러들이 섞여 있어서 부끄러운 경험이었습니다.

이 경험을 통해 에러를 찾는 시각이 넓어졌습니다. '꺼진 불도 다시 보자'는 속담처럼, 오류나 오타가 없는 코드도 꼼꼼히 살펴보고, 어떤 경우에도 속단하지 않고 에러를 찾아가는 자세를 배웠습니다. 덕분에 에러를 찾는 속도가 빨라졌습니다. 물론 현재도 오류를 만나 프로젝트를 완성하지 못한 상태이지만, 오류의 원인을 찾아 원하는 기능을 구현하기 위해 계속 도전 중입니다.
